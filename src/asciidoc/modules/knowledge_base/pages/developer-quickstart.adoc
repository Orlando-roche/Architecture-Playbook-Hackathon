= Developer Quickstart
:page-content-owner-email: jon.fuller@roche.com

This document is aimed to give a quick overview of some of the useful tools and platforms for developers at Roche, so that new joiners can get started developing effectively. Weâ€™d recommend that you take this as a starting point, and ask around if a tool or platform is missing. The goal is to strike a balance between effective personal workflow, and gaining from critical mass working in a similar way.

The Roche Dev Hub gives some information about platforms that you could consider defaults to use. Of course there may be a reason to need something different, but the linked platforms are well documented, well supported and the easiest place to start. When using these tools and platforms you are most likely to find a colleague, or example to help you with any issues arise.

== Windows Sub-system for Linux
On Windows you probably need WSL2. 
Install with the [standard instructions]. 
You may need to do some more nonsense, to get internal connectivity to work as expected, see https://overflow.roche.com/t/external-connectivity-from-wsl-2/7264/26[here]. 
Beware WSL will eat your hard drive, especially when used with Docker! https://rochewiki.roche.com/confluence/pages/viewpage.action?pageId=709528197[See the following to reclaim space]

== Docker

Docker Desktop, and Docker Hub require a license to use. Roche has access to these licenses. The docs are https://penguins.pages.roche.com/solutions-wiki/Docker/docker/[here], and the link to the license application is https://devhub.roche.com/docs/default/System/fk-ef-docker-business[there].

You might prefer to use Rancher as a front-end to docker-ce (docker engine).
