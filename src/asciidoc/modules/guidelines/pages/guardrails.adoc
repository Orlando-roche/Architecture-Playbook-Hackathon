== Governance & Compliance

* An *Architecture Review* must be done by all new or significantly modified systems which: 
.. Have a substantial relevance and impact on the landscape (e.g. new registration systems, platforms)
.. Deviate from the reference as described in this document
* *Decision Transparency:* The Architecture team must document decisions in the Architecture Decision Records (ADRs)
* Computerized systems must perform a *Security and Risk Assessment* link:https://iraam.roche.com/[IRM].
* Register your solution in link:https://act.roche.com/[ACT] if you require the option to *test application compatibility* on client machines. 
* New applications requiring *infrastructure from Global Informatics*, must be registered in link:https://sites.google.com/roche.com/snow-itsm-for-pred[ITSM Service Now].
* Artifacts of *Business Information and Architecture* must be captured in the link:https://go.roche.com/amr[Architecture Master Repository (AMR)].
* Adhere to *local compliance rules* for user-facing applications, e.g. in Germany the link:https://roche.service-now.com/people_and_culture?id=kb_article&sysparm_article=KB0081364[works council approval] is required before making a new tool available to employees. 


== Data, Security & Privacy 

* *Data Classification:* Define and protect data according to Roche privacy and data-handling policies.
* Application defines and uses a *terminology* which is accessible (recommended terminology server is link:https://go.roche.com/rts[RTS])
* link:https://go.roche.com/FAIRArchitecturePlaybook[FAIR] principles are applied.
* *Audit & Monitoring:* Systems must support auditability and operational visibility.
