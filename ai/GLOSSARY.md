# Glossary

Key terms and definitions used in the CS-CoE Architecture Playbook.

---

## Organizations and Teams

| Term | Definition |
|------|------------|
| **CS-CoE** | Computational Sciences Center of Excellence - the unified organization for computation and data ecosystems across Roche's Research and Early Development |
| **pRED** | Pharma Research and Early Development - Roche's pharmaceutical research division |
| **gRED** | Genentech Research and Early Development - Genentech's research division |
| **REDs** | Research and Early Development organizations (pRED and gRED collectively) |
| **DDC** | Digital, Data & Compute - a department within CS-CoE |
| **CICoR** | China Innovation Center of Roche |
| **RDT** | Roche Digital Technology - corporate IT organization |
| **GIS** | Global Informatics Services |

---

## Architecture Roles and Bodies

| Term | Definition |
|------|------------|
| **Architecture Catalyst** | Individual who acts as a bridge between the Architecture Department and the broader organization |
| **Architecture Council** | Body that connects the Core Architecture Team with the broader CS-CoE organization |
| **Architecture Review Forum** | Body that reviews and approves architectural designs, decisions, and changes |
| **Architecture Workgroup** | Time-bound, cross-functional team formed to deliver a specific architecture outcome |
| **Core Architecture Team** | The central architecture group responsible for architectural direction and governance |

---

## Architecture Artifacts

| Term | Definition |
|------|------------|
| **ADR** | Architecture Decision Record - a document capturing a significant architectural decision |
| **AMR** | Architecture Master Repository - pRED's tool for managing architecture artifacts |
| **Archimedes** | Roche's enterprise-wide architecture management tool offered by RDT |
| **Guardrails** | Key boundaries and principles that guide design, implementation, and evolution of solutions |
| **Reference Architecture** | Standard patterns and guidance for building systems |

---

## Platforms and Tools

| Term | Definition |
|------|------------|
| **ACT** | Application Compatibility Testing - tool for testing application compatibility on client machines |
| **Artifactory** | Binary repository manager (eu.repository.roche.com) |
| **Dagster** | Open-source data orchestration platform |
| **DevHub** | Roche's internal developer portal |
| **GitLab** | Source code management platform (code.roche.com) |
| **Gravitee** | API Gateway for registering and publishing APIs |
| **IRM** | Information Risk Management - tool for security and risk assessments (iraam.roche.com) |
| **ITSM** | IT Service Management - ServiceNow for infrastructure requests |
| **Kubeflow** | Open-source ML platform for Kubernetes |
| **OneDoc** | Documentation platform for technical documentation |
| **PingFederate** | Roche's SSO identity provider |
| **RCP** | Roche Cloud Platform - standard cloud infrastructure offering |
| **RTS** | Roche Terminology Server - recommended terminology server |

---

## Data and Integration

| Term | Definition |
|------|------------|
| **API-first** | Design approach where APIs are defined before implementation |
| **FAIR** | Findable, Accessible, Interoperable, Reusable - data principles |
| **GUPRI** | Globally Unique, Persistent and Resolvable Identifier |
| **MCP** | Model Context Protocol - protocol for AI agent integrations |
| **OAuth 2.0** | Authorization framework for secure API access |
| **OIDC** | OpenID Connect - authentication layer on top of OAuth 2.0 |
| **PBAC/PBAM** | Project Based Access Control/Management |
| **SSO** | Single Sign-On - authentication mechanism |

---

## Processes and Governance

| Term | Definition |
|------|------------|
| **Architecture Review** | Formal review of architectural designs by the Architecture Review Forum |
| **BER** | Biological Entity Registration - initiative for managing biological research master data |
| **Exception Process** | Process for requesting deviations from guardrails |
| **Seed-and-grow** | Strategy of starting small and expanding based on proven value |

---
